# Accounting Analytics Dataset

<DatasetName /> is a synthetic accounting analytics dataset built for classes that need more than disconnected sample tables. It combines business context, process flows, source documents, posted `GLEntry` activity, and classroom-ready guidance so users can move from business events into accounting outcomes without rebuilding the model first.

## What Comes With The Dataset

- <ReleaseDownloadLink type="sqlite">SQLite database download</ReleaseDownloadLink> for SQL practice, source-to-ledger tracing, and audit review
- <ReleaseDownloadLink type="excel">Excel workbook download</ReleaseDownloadLink> for pivots, charts, filters, and instructor-led walkthroughs
- <ReleaseDownloadLink type="csv">CSV package download</ReleaseDownloadLink> for table-by-table imports and external analysis tools
- process, analytics, and instructor pages that explain how to use the same dataset across multiple accounting teaching paths

## Where It Fits Best

This dataset works especially well when a course needs one shared business model for:

- AIS and business-process tracing
- accounting analytics and SQL labs
- audit and control walkthroughs
- financial statement, working-capital, and close-cycle analysis
- managerial, payroll, and manufacturing interpretation

The value is not only the files. The value is the link between the files and the business story behind them.

## Why Instructors Keep One Model

<DisplayName /> is intentionally broad enough to support several accounting paths without changing the company or the data package. Students can learn how O2C, P2P, manufacturing, payroll, and finance interact in one environment instead of switching between unrelated exercises or proprietary systems.

That makes the dataset useful when the assignment needs explanation as well as execution. A student can start in a shipment, a payroll register, or a journal entry and still work back to the same business story.
