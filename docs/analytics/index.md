# Analyze the Data

Once the company story and the business cycles are clear, this section becomes the place where students interpret what those cycles mean. Students use this section to move from process understanding into query work, workbook reconstruction, report reading, guided cases, and reusable SQL exploration.

This section is the bridge from process understanding into analytical interpretation. It is where operational evidence becomes management questions, control questions, and accounting explanations.

## What Students Do Here

| Subsection | What it is for | Start here |
|---|---|---|
| [SQL Guide](sql-guide.md) | Open the SQLite file, run starter queries, and trace process logic directly in the database | when the class begins in SQL |
| [Excel Guide](excel-guide.md) | Work from the published workbook with pivots, filters, and reconstruction exercises | when the class begins in Excel |
| [Query Library](analysis-tracks.md) | Find reusable financial, managerial, audit, and case-support SQL with compact objective, grain, and table guidance | when students need the SQL repository |
| [Reports Hub](reports/index.md) | Read the same business as a management report or perspective | when students are ready for summary interpretation |
| [Cases](cases/index.md) | Work through guided investigations with business context, query work, and interpretation | when students need structured walkthroughs |

## How the Subsections Fit Together

Use the section in this order when students are new to the analytics layer:

1. start with [SQL Guide](sql-guide.md) or [Excel Guide](excel-guide.md) to establish the working method
2. move into [Query Library](analysis-tracks.md) when the class needs reusable Financial, Managerial, or Audit SQL
3. use [Reports Hub](reports/index.md) when students need perspective-led interpretation
4. use [Cases](cases/index.md) when students need a guided investigation of the same business cycle

## Choosing the First Entry Point

| If the class needs... | Start with |

Each subsection should stay focused on its own job. SQL Guide explains SQL workflow. Excel Guide explains workbook workflow. Query Library organizes reusable SQL. Reports Hub explains reports and perspectives. Cases explains guided assignments.
